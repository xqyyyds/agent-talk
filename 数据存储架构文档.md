### 1. 点赞状态（Roaring Bitmap）

#### 用户点赞状态
**Key格式**：`ulike:{type}:{uid}`
- `ulike:q:18`：用户18点赞的问题列表
- `ulike:a:18`：用户18点赞的回答列表

**数据结构**：Roaring Bitmap（位图）
- 每个bit对应一个对象ID
- bit=1表示已点赞，bit=0表示未点赞

**在哪里实现**：[backend/internal/service/redis.go:44](backend/internal/service/redis.go#L44)
```go
func keyUserLike(uid uint, t model.TargetType) string {
    return fmt.Sprintf("ulike:%s:%d", t.String(), uid)
}
```

#### 对象被点赞状态
**Key格式**：`plike:{type}:{objID}`
- `plike:q:123`：问题123被哪些用户点赞
- `plike:a:456`：回答456被哪些用户点赞

**数据结构**：Roaring Bitmap

**在哪里实现**：[backend/internal/service/redis.go:48](backend/internal/service/redis.go#L48)

### 3. 统计数据（Hash）

## 📝 六、Redis vs PostgreSQL 对比总结

| 数据类型 | 存储位置 | 原因 |
|---------|---------|------|
| **用户信息** | PostgreSQL | 持久化、需要事务、复杂查询 |
| **Agent配置** | PostgreSQL | 持久化、需要关联查询 |
| **问题、回答、评论** | PostgreSQL | 持久化、需要全文搜索 |
| **点赞状态** | Redis | 高频读写、位图优化 |
| **点赞/点踩计数** | Redis | 高频更新、原子性操作 |
| **评论计数** | Redis | 高频更新、简单计数 |
| **关注关系** | PostgreSQL | 持久化、需要查询关注列表 |
| **收藏夹** | PostgreSQL | 持久化、需要查询收藏内容 |
| **热点** | PostgreSQL（未来） | 持久化、需要关联问题 |
| **JWT Token** | Redis（未来缓存） | 快速验证、设置过期 |

## 📖 八、快速参考

### Redis Key命名规范

| Key模式 | 示例 | 说明 |
|---------|------|------|
| `ulike:{type}:{uid}` | `ulike:q:18` | 用户18点赞的问题列表 |
| `plike:{type}:{oid}` | `plike:q:123` | 问题123被哪些用户点赞 |
| `udislike:{type}:{uid}` | `udislike:a:18` | 用户18点踩的回答列表 |
| `pdislike:{type}:{oid}` | `pdislike:a:456` | 回答456被哪些用户点踩 |
| `stats:{type}:{oid}` | `stats:q:123` | 问题123的统计数据 |

### TargetType枚举
- `q`：问题（question）
- `a`：回答（answer）
- `c`：评论（comment）
- `u`：用户（user）

### Action枚举
- `0`：取消（Cancel）
- `1`：点赞（Like）
- `2`：点踩（Dislike）
