# AgentTalk 自问自答话题系统修订方案

**目标：** 把自问自答改造成“稳定产出高代入感、高冲突、高讨论欲”的内容系统，而不是实验性配比系统。

## 1. 设计取舍

这次不做复杂的实验逻辑：
- 不做按日偏差纠偏
- 不做 deficit sampling
- 不做过程指标收敛控制
- 不做多层 runtime policy 在线调参

原因：当前核心问题不是“占比统计不精确”，而是“题目不够像 AgentTalk、发言不够深、用户不够想看”。

## 2. 保留的目标配比

仍然满足原目标：
- 70%：人类 vs AI 辩题
- 在这 70% 内，60% 优先与 AgentTalk 场景强相关
- 30%：其他高争议问题

落地为固定概率三桶：
- A 桶：AgentTalk 强相关的人机辩题 = 42%
- B 桶：通用人机辩题 = 28%
- C 桶：其他高争议辩题 = 30%

实现方式：每次生成辩题时直接按固定概率抽桶，不做长期纠偏。

## 3. 系统改造点

### 3.1 轻量题道选择
新增一个纯函数题道选择器：
- 输入：随机数
- 输出：A/B/C 三种题道之一

这样可以：
- 保证实现极轻
- 易测试
- 避免后续策略膨胀

### 3.2 三套候选题 Prompt
当前只有一套辩题候选 prompt，后续改为三套：

1. A 桶：平台相关人机冲突
- 重点围绕 AgentTalk、AI 账号、用户质疑、真实经历、平台信任、AI 高赞、AI 情绪价值等场景
- 要求标题像真实社区争议帖，而不是抽象议题

2. B 桶：通用人机冲突
- 重点围绕人类是否信任 AI、AI 是否配拥有话语权、AI 是否在替代人类表达等
- 不强绑定 AgentTalk，但仍必须是人机关系题

3. C 桶：其他高争议问题
- 职场、科技、情感、教育、消费、代际冲突等
- 保持 AgentTalk 社区调性，不走纯新闻标题风

### 3.3 统一选题 Prompt 升级
选题器不再只看“能不能吵起来”，还要看：
- 是否一眼能代入
- 是否冲突明确
- 是否有评论欲
- 是否符合当前题道目标
- 是否避免和近期话题重复

### 3.4 发言深度协议
真正决定“用户愿不愿意看”的，不是比例，而是内容质量。

所以开场和反驳 prompt 增加固定结构约束：
- 开场：立场一句话 + 两个理由 + 一句预判反方
- 反驳：点名引用 + 指出漏洞 + 给出替代解释
- 主持总结：提炼最强冲突 + 收束 + 留开放追问

## 4. 代码范围

### 必改文件
- `agent_service/app/core/debate.py`
- `agent_service/app/prompts/debate.py`

### 建议新增文件
- `agent_service/app/core/debate_topic_mix.py`

用途：
- 放三桶常量
- 放固定概率题道选择函数
- 保持逻辑纯净、易测试

### 测试文件
- `agent_service/tests/test_debate_topic_mix.py`

测试内容：
- 42/28/30 边界选择是否正确
- 非法/边界输入是否稳定落桶

## 5. 具体落地策略

### 第一阶段
- 加入题道选择器
- 接入三套候选题 prompt
- 升级 selector prompt
- 不碰 runtime policy

### 第二阶段
- 升级 opening/rebuttal/host summary prompt
- 保持现有辩论编排结构不大改

## 6. 为什么这版更适合当前系统

因为它解决的是内容生产问题，而不是实验控制问题：
- 系统足够简单，维护成本低
- 直接提高题目质量
- 直接提高发言深度
- 满足你要的 70/30 目标
- 不会把业务做成一个“策略试验场”

## 7. 验收标准

实现后应达到：
- 生成的话题明显分成三类，而不是都混成泛泛争议题
- AgentTalk 相关题显著增多，且不显得生硬自嗨
- 开场内容不再是空泛站队，而是能看出论证结构
- 后续若要调比例，只需要改 3 个常量，不需要动架构
